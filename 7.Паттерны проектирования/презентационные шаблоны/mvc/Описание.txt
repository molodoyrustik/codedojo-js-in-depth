во view есть два типа методов: обработчики событий и методы логики. В оброботчиках не должно быть логики просто делегирует допустим вытаскивает данные и уже передает в метод.
во view есть доступ к model она нужна чтобы отрендирить какие то начальные данные которые уже есть в модели. но во view нельзя обнавлять удалять и редактировать данные через model а только для получения каких то начальных данных.
Ответственность за обнавление данных в model лежит на controllerе и дальше мы можем после обнавления данных попросить view показать какое то сообщение что данные обнавились или удалились и тд.
Controller является Mediatorom между моделью и вью.
с Backendom общается Model
Про связку данных при обнавлении Модели: можно во Вью при иницализации подписаться на событие обновления Модели. Логику связки данных надо сувать в Модель(Event Emitter).
Основная проблемы MVC слишком много знает View то есть имеет доступ и к Controller and Model
